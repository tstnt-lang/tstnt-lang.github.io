// GET STARTED

Install in minutes.

TSTNT runs anywhere Rust compiles. Including your Android phone.

01

Install Rust

pkg install rust

On Termux / Android. On Linux use rustup.rs

02

Clone & Build

cargo build --release

From the tstnt repository

03

Add to PATH

cp target/release/tstnt ~/bin/

Then tstnt --version to verify

04

Write code

tstnt hello.tstnt

Or start the REPL: tstnt repl

Clean. Readable. Fast to write.

TSTNT syntax is designed to be obvious at a glance.

structs & impl
struct User { name: str age: int } impl User { do greet -> str { return "Hi, " + self.name } do is_adult -> bool { return self.age >= 18 } } let u = User { name: "Alex", age: 25 } print(u.greet())
match & loops
match score { 100 -> print("perfect!") 80..99 -> print("great") _ -> print("keep going") } # Enumerate loop loop i, item in fruits { print(i + ": " + item) } # Functional style let evens = filter(nums, |x| x % 2 == 0) let sum = reduce(evens, |acc x| acc + x, 0)
telegram bot
use tg use thread do main { tg.token("YOUR_TOKEN") tg.delete_webhook() let mut offset = 0 while true { let updates = tg.get_updates(offset) loop upd in updates { offset = upd.update_id + 1 tg.send_keyboard(upd.chat_id, "Hello!", [["Start", "Help"]]) } thread.sleep(500) } }

// CHANGELOG

From idea to v1.5.0.

Every version, every step. ∆ marks dev-only builds — never released.

v1.5.0RELEASE
The performance release
  • VM call overhead reduced 6× — vec drain → stack reuse
  • Globals now indexed Vec instead of HashMap — O(1) access
  • ARM64 JIT compiler for pure-int functions
  • New syntax: for x in arr, const declarations
  • New stdlib: iter, bytes, reflect
  • 12 new packages: vec2d, neural, color3, fsm, lru, trie, bitset, diff2, text-wrap, promise, multimap, expr-eval
  • Site moved to tstnt.lol
∆1.4.3DEV
JIT experiments + globals refactor
  • First working ARM64 JIT: add(a,b) compiles to native
  • Globals replaced with indexed Vec — HashMap eliminated
  • Fixed unsafe unwrap_unchecked in VM hot loop
  • Arc → used for shared function chunks across threads
∆1.4.1DEV
VM call stack isolation
  • Function calls no longer pollute caller env stack
  • Flat Vec<(String, Value)> env with scope index stack
  • While loop: single push/pop instead of per-iteration
  • bench4 (1M function calls): 22s → 8s
∆1.3.8DEV
VM drain fix — the big one
  • Discovered: vec.drain().collect() on every call = 1M allocs
  • Fixed: args stay on stack, stack_base = args_start
  • sys time dropped from 4s to 0.037s on bench4
  • bench4: 22s → 1.4s, bench5 (fib): 6s → 4s
∆1.3.2DEV
VM codegen improvements
  • PushIntInline opcode — small ints bypass constant pool
  • unsafe hot path in read_u8, LoadLocal, Return
  • Function params stored as count, not cloned Vec<String>
  • Loop iterators use indexed globals instead of string lookup
∆1.2.5DEV
Bytecode VM shipped internally
  • Full bytecode compiler + stack VM running in production
  • Fallback to tree-walk interpreter when VM can't compile
  • bench3 (pure loop): 1.1s → 0.85s
  • @cache decorator works in VM via CacheCheck/CacheStore opcodes
v1.2.0RELEASE
VM + LSP release
  • Bytecode VM with codegen, opcodes, chunks
  • LSP server for editor support
  • @cache, @timer, @deprecated decorators with real logic
  • New stdlib: sql, http_server, tui2, result, fmt2
  • 30+ new builtins: find, every, any, flat_map, chunks, sprintf…
∆1.1.8DEV
Packages v2 ecosystem
  • Package manager rewritten — install from GitHub
  • 130+ packages written in TSTNT itself
  • tg-rpg, sql-orm, tui-pro, game-2d, neural, algorithms
  • pkg search, pkg install, pkg list commands
v1.1.0RELEASE
The ecosystem release
  • 140+ packages in the ecosystem
  • Telegram bot API stdlib module
